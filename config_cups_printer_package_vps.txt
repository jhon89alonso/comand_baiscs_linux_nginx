Revisar el log de CUPS para identificar errores de filtros, drivers o backend
sudo tail -n 50 /var/log/cups/error_log

Si la impresora ya está configurada pero no imprime, eliminar la cola actual
sudo lpadmin -x NOMBRE_IMPRESORA

Crear nuevamente la impresora usando protocolo socket (Puerto 9100)
Este procedimiento funciona para la mayoría de impresoras de red:
sudo lpadmin -p NOMBRE_IMPRESORA -E -v socket://IP-DE-LA-IMPRESORA:9100 -m raw
Nota: el modelo "raw" evita que CUPS modifique los datos enviados, permitiendo compatibilidad con impresoras de etiquetas (Zebra), impresoras térmicas y modelos que trabajan con su propio lenguaje (ZPL, EPL, ESC/POS, PCL, etc.).

Enviar prueba de impresión en modo RAW
echo "^XA^FO50,50^A0N,60,60^FDRUN RAW OK^FS^XZ" | lp -d NOMBRE_IMPRESORA -o raw
Para impresoras normales (HP, Pantum, Epson), se puede usar un texto simple:
echo "TEST DE IMPRESIÓN" | lp -d NOMBRE_IMPRESORA -o raw

Confirmar que la impresora imprime correctamente
Revisar nuevamente el estado:
lpstat -t

NOTAS GENERALES:

El puerto estándar para impresión por red es el 9100 (Protocolo JetDirect / RAW).

El modo "raw" es el más compatible y evita problemas con drivers incorrectos.

Si la impresora requiere un driver específico (normalmente HP, Epson, etc.), se puede instalar desde CUPS con el modelo adecuado, pero si presenta fallos siempre se puede volver al modo RAW.

Revisar siempre el archivo /var/log/cups/error_log si hay problemas.

Este procedimiento es válido para impresoras láser, térmicas, de etiquetas y multifuncionales, siempre que admitan impresión por red.

SOLUCIÓN IMPLEMENTADA
================================================================================

--------------------------------------------------------------------------------
1. CREACIÓN DE SCRIPT DE RUTAS AUTOMÁTICAS
--------------------------------------------------------------------------------

Archivo: /etc/openfortivpn/route-up.sh

Contenido:
----------
#!/bin/bash
# Esperar a que la interfaz esté completamente lista
sleep 5

# Verificar que ppp0 existe
if ip link show ppp0 &> /dev/null; then
    # Agregar la ruta
    /sbin/ip route add 192.168.169.0/24 dev ppp0 2>/dev/null || true
    # Log para debugging
    echo "$(date): Ruta agregada" >> /var/log/openfortivpn-routes.log
    /sbin/ip route | grep 192.168.169 >> /var/log/openfortivpn-routes.log
else
    echo "$(date): ERROR - ppp0 no existe" >> /var/log/openfortivpn-routes.log
fi
----------

Permisos aplicados:

sudo chmod +x /etc/openfortivpn/route-up.sh
sudo touch /var/log/openfortivpn-routes.log
sudo chmod 666 /var/log/openfortivpn-routes.log
